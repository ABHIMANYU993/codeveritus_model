# Codeveritus - AI vs Human Code Classifier

## Executive Summary
Codeveritus is a state-of-the-art research and deployment codebase designed to address the challenges of distinguishing between computer code authored by human developers and code generated by Large Language Models (LLMs) such as GPT-4, Claude, and Gemini. Leveraging advanced Natural Language Processing (NLP) models, specifically pre-trained CodeBERT architectures, Codeveritus provides a complete end-to-end framework. This framework covers raw dataset ingestion, processing pipelines, model training and fine-tuning, evaluation, multiple server deployments, and a modern React-based web user interface.

## Project Goals
1. Develop a high-accuracy, robust classification model utilizing deep learning transformer architectures.
2. Provide data preprocessing and tokenization scripts tailored for source code structure, syntax, and keywords.
3. Support multiple deployment patterns including Django backend servers, Flask prototypes, FastAPI microservices, and Hugging Face spaces.
4. Deliver an enterprise-grade production-ready system utilizing Express.js backends and React frontends deployed on Alpine Linux VMs via Docker and Nginx.

## Subsystem Breakdowns

### 1. Model Training & Preprocessing (`model/training`)
The machine learning training subsystem uses PyTorch and Hugging Face Transformers. The primary model architecture is built on top of `microsoft/codebert-base`. CodeBERT is chosen because it is pre-trained on bimodal data (natural language and source code), allowing it to effectively capture the structural semantics of programming languages. The training pipeline includes:
- Slicing code snippets and parsing them into specialized tokenizers.
- Training using PyTorch's native loops with gradient accumulation to accommodate larger batch sizes on consumer-grade hardware.
- Exporting the trained model weight arrays into the high-performance, secure `.safetensors` format rather than traditional pickle-based `.bin` or `.pt` formats.

### 2. Back-end Service (`back-end`)
The production API service is built using Node.js and Express. It connects to a MongoDB database to persist user registrations, admin accounts, and code analysis submissions. Security is managed through JSON Web Token (JWT) signatures. Key features include:
- Secure authentication APIs for users and administrators.
- Submissions tracking: Stores code snippets, classification probabilities, and timestamps.
- Automated middleware for handling Cross-Origin Resource Sharing (CORS) and validating authorization headers.

### 3. Front-end Client (`front-end`)
The user interface is a responsive Single Page Application (SPA) built using React. It features high-quality assets (such as background looping videos and custom transparent logos) and dynamic styling to guide the user. Key screens include:
- **Home Page**: Introduces the project's use case, showing stats, features, and call-to-actions.
- **Login/Registration**: Simple, secure forms to access personalized dashboards.
- **Code Analysis Interface**: A text field allowing users to paste code snippets and trigger classifier inferences.
- **Results Dashboard**: Displays confidence scores, token distributions, and logs of prior submissions.

## Production VM Initialization and Execution Guide
For optimal performance and security, the system is designed to run in isolated Docker containers deployed on an Alpine Linux virtual machine. Below are the steps to spin up the entire system from scratch.

### Prerequisites
Before setting up Codeveritus, ensure your Alpine Linux system has Docker, Docker Compose, and Git installed and running. You can install them using Alpine's package manager (`apk`):
```bash
apk update
apk add docker docker-compose git git-lfs
rc-update add docker boot
service docker start
```

### Step 1: Clone the Repository and Pull Git LFS Weights
Because the model weights are stored in LFS, you must pull them explicitly:
```bash
git clone <your-repository-url> codeveritus
cd codeveritus
git lfs install
git lfs pull
```

### Step 2: Configure Environment Variables
You must configure the env files for both backend and frontend systems.

**For the Backend (`back-end/.env`):**
```env
PORT=5000
MONGODB_URI=mongodb://mongodb:27017/codeveritus
JWT_SECRET=super_secure_random_hash_key_here
MODEL_SERVICE_URL=http://fastapi_server:8000/predict
```

**For the Frontend (`front-end/.env`):**
```env
REACT_APP_API_URL=http://<your-vm-ip-or-domain>:5000/api
```

### Step 3: Build and Run with Docker Compose
In the root of the project, create a unified `docker-compose.yml` file to orchestrate the backend, frontend, database, and FastAPI model classifier:
```yaml
version: '3.8'
services:
  mongodb:
    image: mongo:6.0
    container_name: codeveritus_db
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db
    networks:
      - codeveritus_net

  fastapi_server:
    build: ./model/deployment/fastapi_server
    container_name: codeveritus_model_server
    ports:
      - "8000:8000"
    volumes:
      - ./model/training/primary_model.safetensors:/app/model.safetensors:ro
    networks:
      - codeveritus_net

  backend:
    build: ./back-end
    container_name: codeveritus_backend
    ports:
      - "5000:5000"
    environment:
      - PORT=5000
      - MONGODB_URI=mongodb://mongodb:27017/codeveritus
      - JWT_SECRET=super_secure_random_hash_key_here
      - MODEL_SERVICE_URL=http://fastapi_server:8000/predict
    depends_on:
      - mongodb
      - fastapi_server
    networks:
      - codeveritus_net

  frontend:
    build: ./front-end
    container_name: codeveritus_frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - codeveritus_net

volumes:
  mongo_data:

networks:
  codeveritus_net:
    driver: bridge
```

Start all services in detached mode:
```bash
docker-compose up --build -d
```

### Step 4: Verify Deployment Status
Check that all Docker containers are running successfully:
```bash
docker ps
```
If you need to view logs for troubleshooting, run:
```bash
docker-compose logs -f backend
```

## Nginx Configuration for Frontend
The frontend Dockerfile uses a multi-stage build. First, it runs `npm run build` to compile static assets. Next, it serves those assets via Nginx running inside Alpine Linux. The custom Nginx configuration (`front-end/nginx.conf`) handles React router requests correctly by routing all missing routes back to `index.html`:
```nginx
server {
    listen 80;
    server_name localhost;
    location / {
        root /usr/share/nginx/html;
        index index.html index.htm;
        try_files $$uri $$uri/ /index.html;
    }
    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /usr/share/nginx/html;
    }
}
```

## CI/CD Deployment Workflow
A typical CI/CD flow on Alpine Linux incorporates automated webhooks connected to GitHub. When changes are pushed to `master`:
1. A runner triggers the pull script.
2. Docker Compose rebuilds the updated service container.
3. A zero-downtime container switch is performed using Nginx upstream routing.

## API Specifications Reference

### Authentication Endpoints

#### POST `/api/users/register`
- Description: Registers a new developer account.
- Request Body:
  ```json
  {
    "username": "johndoe",
    "email": "john@example.com",
    "password": "securepassword123"
  }
  ```
- Response (201 Created):
  ```json
  {
    "message": "User registered successfully.",
    "userId": "65c2b3e8a71d2b1f0c2a5d9a"
  }
  ```

#### POST `/api/users/login`
- Description: Authenticates user credentials and returns a JWT.
- Request Body:
  ```json
  {
    "email": "john@example.com",
    "password": "securepassword123"
  }
  ```
- Response (200 OK):

### Submission Endpoints

#### POST `/api/submissions/analyze`
- Description: Accepts a code snippet for analysis, sends it to the FastAPI model microservice, and records the results.
- Headers: `Authorization: Bearer <JWT_TOKEN>`
- Request Body:
  ```json
  {
    "codeSnippet": "def calculate_sum(a, b):\n    return a + b\n",
    "language": "python"
  }
  ```
- Response (200 OK):
  ```json
  {
    "submissionId": "65c2c512a71d2b1f0c2a5d9e",
    "prediction": "human",
    "confidence": 0.9854,
    "timestamp": "2025-02-18T14:46:50.000Z"
  }
  ```

#### GET `/api/submissions/history`
- Description: Retrieves history of code analyses performed by the authenticated user.
- Headers: `Authorization: Bearer <JWT_TOKEN>`
- Response (200 OK):
  ```json
  [
    {
      "id": "65c2c512a71d2b1f0c2a5d9e",
      "prediction": "human",
      "confidence": 0.9854,
      "language": "python",
      "timestamp": "2025-02-18T14:46:50.000Z"
    }
  ]
  ```
